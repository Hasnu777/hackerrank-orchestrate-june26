## Strategy Comparison

Two strategies were considered:

### Strategy A — Multi-stage Claude pipeline (final strategy used)

Each claim is processed in three sequential Claude API calls:
- **Stage 1 (history)**: Text-only credibility assessment from user claim history
- **Stage 2 (vision)**: Image analysis — damage type, severity, evidence standard
- **Stage 3 (decision)**: Synthesis of history and visual results into final verdict

**Pros**: specialised prompts per stage, visual and text reasoning decoupled, each stage can use the most cost-effective model  
**Cons**: three API calls per claim, higher latency than a single-pass approach

### Strategy B — Single-pass VLM (considered, not used)

All context (images, history, evidence requirements) sent in one prompt. Simpler but the model must handle history risk and visual analysis simultaneously, reducing the quality of each sub-task.

**Decision**: Strategy A was chosen for accuracy and output quality.
